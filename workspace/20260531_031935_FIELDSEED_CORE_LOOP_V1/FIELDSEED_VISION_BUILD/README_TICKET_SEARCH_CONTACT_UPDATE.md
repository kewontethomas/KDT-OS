# Ticket Search + Contact Update

Changed:
- Ticket ID is optional, not required.
- Added Search Tickets page.
- Search by site, contact person, phone, email, issue words, system, access method, notes, next action, etc.
- Added contact person to ticket creation.
- Added contact person to Collector missions.
- Open Tickets still lets you select by ID once you find it.
- The database updates itself when tickets are created, searched, updated, and closed.

Recommended workflow:
1. Create ticket naturally.
2. Add site/contact if you know it.
3. Later find it with Search Tickets using site/contact/issue words.
4. Open by ticket # from search results.
5. Add findings or close + learn.
